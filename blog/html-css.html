What does it mean to display inline vs inline-block?

When we are writing content for a website, we wrap text in various tags to label it and give it different significance. Each element appears within a rectangular box and has its own display properties by default. (These rectangular boxes can be made visible by changing the value of the border property of each element). Understanding the way that different tags behave when a web page is loaded is very useful when it comes to designing the layout of your website.

Inline elements

Inline elements like <a>, <span>, <img> and <button> have a number of default properties to be aware of when considering using them in your website:

  • They take up as much space as needed
  • They can be placed side by side, in the natural flow of the web page
  • You can not set the width and height of these elements

Block-level elements

Block-level elements like <div>, <p>, <h1>, <ul> and <footer> have default properties of their own and also have their own limitations. These properties include:

  • They will start on a new line
  • By default, they will take up the full width of the page
  • You can set the width and height of these elements
